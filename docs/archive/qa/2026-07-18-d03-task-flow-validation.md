## 2. 本轮修复

执行 D-03 前发现两个会阻塞验收的缺口：

- 看板卡片只能点击进入详情，不能在状态列之间移动。
- 子任务列表只支持一层展示和新增，无法完成 D-01 P0-08 要求的三层子任务验收。

已完成修复：

- 为看板状态列增加拖放目标，任务卡片支持拖拽移动到 `待收集`、`待处理`、`进行中`、`待验收`、`已关闭`。
- 拖放后更新任务 `status`、`is_completed` 和 `progress`，并显示移动成功提示。
- 子任务改为递归渲染，每一层子任务都可继续新增下级子任务。
- 父任务完成时，使用递归后代更新，关闭所有层级的子任务，避免父子状态不一致。
