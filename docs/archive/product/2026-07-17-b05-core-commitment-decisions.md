## 2. Dashboard 处理

当前 Dashboard 保留数据概览、今日任务、最近笔记、最近视频、统计和番茄钟入口。卡片拖拽排序与用户自定义固定工具不进入 RC 验收门禁，原因是两者需要持久化布局模型，并且会影响备份、恢复和多用户隔离。

RC 文案建议使用：

> Dashboard 提供本地任务、笔记、书籍、视频和专注工具的快捷概览。

避免使用：

> Dashboard 支持自由编排和用户自定义固定小工具。

证据：`src/views/Dashboard.tsx`、`src/components/Sidebar.tsx`。

## 3. 任务处理

### 3.1 自然语言

保留当前快速添加入口，但把能力描述限制为：

- 支持有限的优先级 token。
- 支持有限的日期 token。
- 其他自然语言不保证解析准确性。

不在 RC 承诺“明天下午 3 点”等复杂中文时间表达的稳定识别。完整 NLP 需要单独定义解析器、时区、歧义确认和失败回退。

### 3.2 子任务

当前 `parent_id` 模型和子任务交互可以作为 RC 的基础能力，但 D-03 必须验证：

- 至少三层子任务创建与展示。
- 父任务完成时子任务状态策略是否一致。
- 子任务编辑、删除、重新打开不破坏父子关系。
- 备份恢复后父子关系仍保持。

在完成上述验收前，发布说明使用“子任务”而不是“无限层级项目树”。

### 3.3 提醒

当前周期规则由 `electron/main.ts` 后台调度，任务页提供规则编辑。原生通知、提前 5/15/60 分钟、自定义提前时间和托盘未完成数尚未形成完整证据，因此本轮不拆出新实现阻塞 B-05，而放入 D-08 真实场景验收。

## 4. 笔记处理

保留 Markdown、笔记本组织、基础导出和已有 `[[标题]]` 引用路径。以下能力不作为当前 RC 完整承诺：

- 完整双向链接索引、反向链接导航和冲突处理。
- 30 天回收站与恢复。
- 版本历史快照。
- 富文本、代码、手绘、表格等多编辑器。

D-04 只验收 Markdown 创建、编辑、预览、删除、导出、重启后保留和基础引用行为。

证据：`src/views/Notes.tsx`、`src/views/NotebookSidebar.tsx`、`tests/` 中的笔记结构与导出相关测试。

## 5. RC 验收影响

B-05 的结论将以下事项从当前 RC 发布阻塞项中移除：

- Dashboard 拖拽重排。
- 用户自定义固定工具。
- 完整自然语言解析。
- 完整 30 天回收站。
- 完整双向链接知识图谱。

以下事项仍保留为真实场景验收门禁：

- 多级子任务的稳定性和备份恢复一致性。
- 周期规则和逾期提醒的真实行为。
- Markdown 笔记与基础引用的重启保留。
