# B-02 LifeOS v1.0 RC 范围基线

日期：2026-07-17
任务：B-02
状态：已完成
依据：B-01 规格—实现—验收矩阵

## 1. 范围决策

当前版本定位为 **LifeOS v1.0 RC 内部验收候选版**，不是完整的“全功能个人操作系统”正式版。RC 的目标是验证本地个人工作台的核心数据闭环、安全性和可恢复性，不继续扩大模块数量。

## 2. 纳入 RC 的核心范围

| 范围 | RC 承诺口径 | 退出条件 |
|---|---|---|
| 账户与桌面壳 | 本地多用户 Profile、登录、退出、语言和主题切换 | D-02 账户与首次启动旅程通过 |
| 任务 | 今日/清单/看板/日历、创建编辑、状态、优先级、截止日期、重复规则、基础子任务 | D-03 任务旅程通过；自然语言只按有限解析描述 |
| 笔记 | 笔记本组织、Markdown 编辑与预览、基础删除、导出 | D-04 笔记旅程通过；不承诺多编辑器和版本历史 |
| 书库 | 已验证的 EPUB/PDF 等格式导入、分类、阅读、TOC、进度、基础批注 | D-05 使用真实 EPUB/PDF 通过 |
| 视频 | 已验证的 YouTube/Bilibili 解析、收藏、下载、队列、失败诊断和本地播放 | D-06 完成真实可访问源和失败场景 |
| 工具箱 | 番茄钟、单位换算、真实加密密码库及迁移 | 密码库安全检查、D-08 异常场景通过 |
| 数据保护 | 手动 ZIP 备份、manifest 校验、当前用户恢复、失败回滚 | A-08/A-09 代码级通过，D-08 完成打包应用演练 |
| 发布基础 | macOS 构建、启动、安装包和更新入口 | C-01 至 C-05 完成真实发布演练后才可转正式发布候选 |

## 3. 降级描述但保留实现

以下能力可以留在代码中，但发布说明、界面提示和产品介绍必须使用受限描述：

- 任务自然语言：描述为“有限快捷解析”，不描述为完整 NLP。
- 汇率工具：描述为“本地换算工具/缓存数据”，不描述为实时汇率服务。
- Dashboard 固定工具：描述为“快捷入口”，不描述为用户自定义 Pin 系统。
- 视频平台：只列出当前有测试和失败诊断证据的平台。
- 全局搜索：当前文案限定为已实现的任务、笔记、书籍范围，或由 B-04 决定补齐视频后再扩大描述。
- 备份：描述为“手动备份与恢复”，不描述为自动增量备份服务。

## 4. 明确移出 v1.0 RC

| 能力 | 决策 | 后续方向 |
|---|---|---|
| 云同步 / OneDrive / WebDAV | 延期 | 先完成本地恢复、冲突策略、密钥管理和服务端边界设计 |
| AI 助手 | 延期 | 另行评估本地 LLM、隐私、成本和离线能力 |
| 移动端 App | 延期 | RC 后重新做跨端数据模型与同步方案 |
| 插件市场 / 第三方工具 | 延期 | 先定义插件权限、沙箱、签名和版本兼容 |
| 网页剪藏 | 延期 | 与笔记导入、浏览器扩展和同步链路一起设计 |
| 日历集成 | 延期 | 先确定 iCal、Google Calendar 和权限模型 |
| 系统托盘任务数 | 延期 | 作为桌面平台增强任务，不阻塞核心数据闭环 |
| Dashboard 拖拽编排 | 延期 | 与用户偏好持久化、布局迁移一起设计 |
| 多格式笔记编辑器 / Git 历史 | 延期 | 拆分为编辑器和版本控制专项任务 |
| 自动备份保留 N 份 / 增量备份 | 延期 | 在最小备份恢复稳定后再优化大文件体验 |
| Windows / Ubuntu 发布级验收 | 延期 | RC 先完成 macOS 主平台，再做跨平台补测 |

## 5. 范围变更规则

- 新增能力必须同时提交负责人、验收标准、风险和对 RC 排期的影响。
- P0 安全、数据损坏和发布阻塞项可以打断计划；其他新增需求默认进入后续版本。
- 任何未达到 B-01 “已完成”标准的能力，不得仅因为 UI 入口存在就标记为完成。
- 若正式发布文案需要扩大承诺，必须先更新 B-01 矩阵和本范围基线，再修改 UI 文案与测试。

## 6. 影响评估

本任务不新增运行时代码，不改变用户数据格式和现有行为。它收敛的是对外承诺和后续排期：

- 减少 RC 需要同时验证的功能面。
- 将云同步、AI、移动端、插件和复杂个性化能力从发布门禁中移除。
- 保留已存在的探索性代码，但要求在发布说明中降级或标记。
- 将真实更新链路、打包验收和数据恢复演练保持为 RC 必须完成的门禁。
