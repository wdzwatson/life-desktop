## 一、结论摘要

本需求可以在现有 LifeOS Electron 架构中实现，建议作为工具箱中的第四个独立插件交付，产品名称暂定为“AI 对话”。

该功能不应只实现一个直接请求模型 API 的聊天界面，而应建立一套可扩展的本地 AI 运行层，统一处理：

- 文本模型的流式对话。
- 用户自定义模型供应商、Agent 与 MCP 服务。
- MCP 工具发现、调用、授权和结果回传。
- 生图、视频生成等异步媒体任务。
- 图片与视频的本地缓存、持久化和安全播放。
- 会话历史、任务恢复、错误诊断和敏感凭据保护。

推荐第一版采用“LifeOS 内置 Agent 运行器”。用户负责填写供应商、模型、系统提示词和 MCP 配置，LifeOS 负责对话循环、工具调用和消息展示。外部 Agent API 作为后续兼容能力，不纳入首版承诺。

## 二、项目背景与现状

### 2.1 需求背景

用户希望在工具箱内增加一个可配置 AI 对话插件，并满足以下核心诉求：

- 可以与 AI 正常进行多轮、流式对话。
- 模型供应商、Agent 行为和 MCP 服务由用户自行填写。
- Agent 可以发现并调用用户配置的 MCP 工具。
- 调用 Grok/xAI 或其他生图、视频 API 时，结果能直接显示在对话中。
- 图片和视频在应用重启后仍能正常访问，而不是依赖短期有效的远程链接。

### 2.2 当前工程基础

现有项目已经具备以下可复用能力：

- React、TypeScript、Vite 与 Electron 桌面壳。
- `contextIsolation` 与 preload IPC 安全桥接。
- 按用户隔离的 SQLite 数据库目录。
- 密码保险箱及加密服务基础。
- 本地视频目录、受控播放 URL 与原生 `<video>` 播放能力。
- `marked` 与 DOMPurify 组成的 Markdown 安全渲染链路。
- yt-dlp / FFmpeg 工具状态管理，可复用于媒体检测、转码和缩略图生成。
- 中文与英文国际化框架，以及四套主题 token。

### 2.3 当前缺口

现有项目尚不具备：

- AI 模型供应商适配层。
- 流式聊天 IPC 协议。
- Agent 工具调用循环。
- MCP Client 与 MCP 服务生命周期管理。
- AI 会话、消息、工具调用和媒体任务数据模型。
- API Key、MCP Token 等专用凭据管理。
- AI 图片和视频的本地资源协议。
- 远程媒体下载、异步任务轮询和恢复机制。
- 面向长会话的虚拟列表、运行状态和取消控制。

## 三、产品目标

### 3.1 核心目标

1. 用户可以在不修改代码的情况下配置模型、Agent 和 MCP。
2. 用户可以完成稳定的多轮流式对话，并管理历史会话。
3. Agent 可以在明确权限边界内调用 MCP 工具。
4. 图片和视频生成结果以原生消息内容显示，并可在本地长期访问。
5. 所有敏感凭据、工具权限和本地文件访问遵循桌面应用安全边界。
6. 供应商差异由适配层吸收，不把供应商原始响应直接暴露给页面。

### 3.2 产品成功标准

- 完成配置后，用户可在 3 分钟内发出第一条有效消息。
- 在供应商正常响应时，文本首段能够持续流式显示，不等待完整响应结束。
- 应用重启后，会话文本、本地图片和本地视频仍可打开。
- MCP 工具调用过程可查看、可授权、可取消、可追溯。
- 单个供应商或 MCP 服务故障不会导致整个应用崩溃。
- 用户能够明确区分“模型回答”“工具结果”“媒体生成任务”和“系统错误”。

### 3.3 非目标

首版不承诺以下能力：

- 自建模型训练、微调或向量数据库平台。
- 多 Agent 自动协作与复杂工作流编排。
- 云端同步会话与凭据。
- 插件市场、MCP 市场或第三方配置自动下载。
- 供应商计费管理、充值和账单对账。
- 绕过模型、内容平台、DRM 或账号授权限制。
- 任意网页 HTML 在聊天中直接执行。
- 无限制的后台任务、无限上下文或无限媒体缓存。

## 四、目标用户与主要场景

### 4.1 目标用户

#### 高级个人用户

- 已拥有一个或多个 AI API Key。
- 希望统一使用不同模型和生图、视频服务。
- 能理解 Base URL、模型名和 MCP 等基本概念。
- 重视本地历史、隐私和配置自主权。

#### MCP 工具用户

- 已部署远程 MCP，或本地安装了 stdio MCP 服务。
- 希望 AI 在对话中读取资料、查询系统或执行自动化。
- 需要明确的权限确认和调用记录。

#### 内容创作者

- 需要在一次会话里完成创意讨论、提示词调整、生图和视频生成。
- 希望直接预览、放大、播放、保存和定位媒体文件。

### 4.2 关键使用场景

1. 配置 OpenAI-compatible 文本模型后开始普通对话。
2. 创建一个带系统提示词和特定 MCP 工具的 Agent。
3. Agent 查询 MCP 工具列表，调用只读工具并继续回答。
4. 高风险 MCP 操作触发用户确认，用户允许或拒绝。
5. 用户要求生成图片，消息中先显示任务进度，完成后显示图片。
6. 用户要求生成视频，任务异步轮询，完成后本地播放。
7. 临时下载 URL 过期，但本地缓存仍可正常打开。
8. 供应商限流、MCP 崩溃或网络中断后，用户获得可执行的恢复建议。
9. 应用重启后恢复未完成媒体任务，或明确标记为可重试。

## 五、范围与优先级

### 5.1 首版必须交付

- 工具箱“AI 对话”入口。
- 首次配置引导和日常对话工作区。
- 自定义模型供应商配置。
- 自定义 Agent 配置。
- MCP Streamable HTTP / SSE 配置与连接测试。
- MCP stdio 基础配置与连接测试。
- 文本多轮对话与流式输出。
- 会话创建、重命名、删除和历史加载。
- 消息 Markdown、安全代码块、复制和重试。
- 工具调用展示、授权、超时和取消。
- 图片生成结果展示与本地缓存。
- 视频生成结果展示、进度、下载和本地播放。
- 运行停止、网络错误、限流错误和配置错误提示。
- 配置、会话和媒体按 LifeOS 用户隔离。
- 中英文文案、键盘操作和基础无障碍。

### 5.2 首版增强项

- 用户向模型上传图片和普通文件。
- 对话内图片画廊和全屏查看。
- 视频封面、时长探测与必要格式转码。
- Agent/MCP 快速切换。
- 单次会话的临时 Agent 参数覆盖。
- 会话导出为 Markdown，并附带媒体文件引用。
- 媒体缓存容量统计和手动清理。

### 5.3 后续版本

- 外部 Agent API 兼容。
- 多 Agent 编排。
- MCP OAuth 完整授权流程。
- 多供应商故障切换和模型路由。
- 知识库、向量检索和笔记/书库深度联动。
- 对话分支、对比回答和多人共享。
- 图片编辑、视频二次剪辑和时间线能力。
- 使用量与费用估算仪表盘。

## 六、配置模型

配置中心分成“供应商”“Agent”“MCP 服务”三层。三层职责必须分离，避免把模型、提示词和工具连接信息混成一个难以复用的配置表单。

### 6.1 模型供应商

模型供应商必须采用独立列表管理，不得只作为 Agent 表单中的临时选项。列表用于统一维护 LifeOS 已接入的文本、图片和视频 API，并向 Agent 提供稳定的供应商引用。

供应商列表应提供：

- 按名称搜索。
- 按协议类型、能力类型和启用状态筛选。
- 显示文本、图片、视频、工具调用和流式输出等能力标识。
- 显示连接状态、最近测试时间和默认模型摘要。
- 创建、编辑、复制、启用、停用和删除操作。
- 设置默认文本、图片或视频供应商；同一能力只能有一个默认项。
- 删除或停用前展示受影响的 Agent，并阻止产生无效引用。
- 支持对单个供应商执行连接测试，不在列表中暴露完整凭据。

每个供应商配置包含：

- 名称。
- 协议类型。
- Base URL。
- API Key 或认证 Token。
- 默认请求头。
- 文本模型名称。
- 图片模型名称，可选。
- 视频模型名称，可选。
- 请求超时。
- 是否允许访问自签名或本地服务，默认关闭。
- 连通性测试状态和最近测试时间。

首版协议类型建议包含：

- OpenAI-compatible 文本对话。
- xAI/Grok 专用适配器。
- 通用自定义 HTTP 适配器，仅在提供明确字段映射后启用。

文本聊天可以优先兼容标准格式，但图片和视频接口不得假定完全兼容。每种媒体供应商需要独立适配请求、任务轮询和结果解析。

### 6.2 Agent

每个 Agent 配置包含：

- 名称和描述。
- 系统提示词。
- 使用的供应商和文本模型。
- 可用 MCP 服务范围。
- 可用工具白名单或黑名单。
- 工具审批策略。
- 最大连续工具调用次数。
- 最大历史消息数或上下文窗口策略。
- 温度、最大输出等模型参数。
- 图片和视频生成使用的供应商/模型。
- 是否为默认 Agent。

工具审批策略：

- 所有工具均确认。
- 只读工具自动执行，写入或命令工具确认。
- 指定工具自动执行。
- 全部自动执行。该模式必须展示风险提示，不作为默认值。

### 6.3 MCP 服务

每个 MCP 配置包含：

- 名称和说明。
- 启用状态。
- 传输方式：Streamable HTTP、SSE 或 stdio。
- HTTP URL、请求头和认证信息。
- stdio 命令、参数、工作目录和环境变量。
- 启动和调用超时。
- 允许的本地目录，可选。
- 最近连接状态、协议版本和工具数量。
- 工具风险级别覆盖设置。

连接测试必须展示：

- 是否成功建立连接。
- MCP 协议协商是否成功。
- 发现的工具数量。
- 资源或提示词能力是否可用。
- 失败阶段与可执行诊断。

## 七、体验与视觉设计方案

### 7.1 设计预检

首次进入且尚未配置时，使用带 AIDA 节奏的引导态；完成配置后切换为高效率三栏对话工作区。动效仅用于首次引导和状态转场，不能阻碍持续聊天、滚动和键盘操作。

Python RNG: components=[Inline Typography Images, Horizontal Accordions, Infinite Marquee]
Python RNG: gsap=[Scroll Pinning, Card Stacking]

AIDA Check: 工具箱导航保留；Attention 为首次进入的宽幅欢迎区；Interest 为配置能力 Bento；Desire 为固定说明与滚动配置卡片；Action 为“测试连接并开始对话”。

Hero Math Verification: 欢迎标题使用 `max-w-6xl` 语义宽度和响应式字号，中文桌面宽度下限制为 2 行；不使用印章图标、标签堆叠、统计数据或无意义徽章。

Bento Density Verification: 桌面引导区采用 6 列乘 2 行，共 12 个网格单元；供应商卡占 3×2=6，Agent 卡占 3×1=3，MCP 卡占 3×1=3，总计 12，无空单元；实现时必须启用 dense flow。

Label Sweep & Button Check: 不使用“SECTION 01”“QUESTION 05”等元标签；主按钮采用强调色背景与可读前景色，次按钮在浅色和深色主题下均达到清晰对比。
</design_plan>

### 7.2 字体与视觉语言

- AI 插件新增视觉层优先使用 Outfit 字体栈；字体未安装或未打包时回退到系统无衬线字体。
- 继续使用 LifeOS 主题 token，不能写死只适用于浅色主题的颜色。
- 强调色用于运行状态、当前 Agent、发送按钮和授权主操作。
- 错误、警告和成功沿用全局 danger、warn、success token。
- 工具调用和系统事件采用更克制的面板，不与模型回答争夺视觉层级。
- 图片和视频使用完整画面，不做无意义装饰性裁切。

### 7.3 首次进入引导态

首次进入且没有可用供应商时，页面按以下顺序呈现：

#### Attention

- 宽幅两行内标题，明确表达“连接你的模型与工具，在一个本地工作台内完成对话和创作”。
- 两个高对比操作：“配置第一个供应商”“导入已有配置”。
- 背景使用主题化径向光晕和轻量颗粒，不使用与功能无关的大型图库图片。

#### Interest

- 三块无空隙 Bento：模型供应商、Agent、MCP 服务。
- 卡片展示配置状态和下一步，不展示虚构统计。
- 能力标题中可使用小型内联媒体缩略图，但不得降低文字可读性。
- MCP 传输方式说明使用横向手风琴展开，减少一次性信息负担。

#### Desire

- 左侧固定“配置后将发生什么”的简短说明。
- 右侧配置卡片依次滚动，使用轻量 Scroll Pinning。
- 已完成的配置卡片以受控 Card Stacking 收拢，保持进度感。
- 尊重系统“减少动态效果”设置，关闭 pin、scrub 和堆叠动画。

#### Action

- 完成供应商、Agent 和至少一个可选 MCP 配置后，提供“测试连接并开始对话”。
- 若用户不需要 MCP，可以明确选择“暂不连接工具”，不能强制配置。

### 7.4 日常对话工作区

配置完成后，默认进入生产力优先的三栏布局：

| 区域 | 建议宽度 | 主要内容 |
| --- | ---: | --- |
| 会话栏 | 220–240px | 新建对话、搜索、会话列表、置顶、更多菜单 |
| 对话区 | 自适应 | 消息时间线、运行状态、输入区、媒体和工具结果 |
| 运行检查器 | 300–320px | Agent、模型、MCP 状态、上下文、工具调用详情 |

顶部局部导航包含：

- 当前会话标题。
- 当前 Agent 选择器。
- 模型状态。
- MCP 连接状态。
- 配置入口。
- 收起或展开运行检查器。

输入区包含：

- 自动增高文本输入框。
- 附件入口。
- 图片生成或视频生成模式入口，默认仍允许自然语言触发。
- 发送/停止按钮。
- 当前 Agent、模型和可用工具摘要。
- 明确的键盘提示：Enter 发送，Shift+Enter 换行。

### 7.5 响应式规则

LifeOS 主窗口最小宽度为 800px，AI 插件必须按可用内容宽度响应：

- 可用宽度大于等于 1180px：三栏同时显示。
- 960–1179px：会话栏与对话区显示，运行检查器改为抽屉。
- 800–959px：以对话区为主，会话栏和检查器均使用抽屉。
- 窄宽度下输入区不得被固定按钮挤压，媒体按单列显示。
- 不允许页面级横向滚动；长代码和工具结果仅在自身容器内横向滚动。

### 7.6 动效原则

- 首次引导可以使用 GSAP ScrollTrigger 的 pin 和卡片堆叠。
- 日常聊天只保留短促的进入、展开、任务进度和停止反馈。
- 新消息不得通过大幅位移动画打断阅读位置。
- 图片和可点击媒体具备轻微缩放反馈，容器必须裁切溢出。
- 视频播放、文本选择、代码复制和长会话滚动期间禁用装饰动画。
- 系统开启“减少动态效果”后，所有非必要动效降级为透明度切换或即时状态变化。

## 八、信息架构与导航

### 8.1 工具箱入口

- 工具箱现有页签增加“AI 对话”。
- AI 对话逻辑和样式必须拆分为独立模块，避免继续扩大现有单文件组件。
- 工具箱切换到其他插件时，正在运行的对话任务可在后台继续，但状态栏必须提示。
- 用户切换 LifeOS 用户时，停止或隔离旧用户的活动任务和 MCP 会话。

### 8.2 AI 插件内部导航

- 对话。
- 模型供应商列表与配置。
- Agent 配置。
- MCP 服务。
- 媒体与存储。
- 运行诊断。

配置页面可以采用检查器/对话框，不需要全部暴露为顶层页签。日常使用时应优先保留对话空间。

## 九、消息内容规范

所有供应商和 MCP 输出必须转换为统一的“消息 + 内容块”结构。页面不得根据供应商名称编写独立消息气泡。

### 9.1 消息角色

- 用户。
- Assistant。
- 工具。
- 系统事件。

### 9.2 内容块类型

- 纯文本。
- Markdown。
- 代码块。
- 图片。
- 视频。
- 音频，预留。
- 普通文件。
- 工具调用。
- 工具结果。
- 媒体生成任务。
- 错误与诊断。

### 9.3 文本消息

- 支持流式增量更新。
- 支持 Markdown、表格、引用、列表和代码块。
- HTML 必须清理，脚本、iframe、object 和事件处理属性一律禁止。
- 消息完成后支持复制、重试和从该消息继续。
- 流式过程中保持稳定滚动；用户主动上滚后停止强制跟随底部。

### 9.4 图片消息

- 支持远程 URL、Base64 和 MCP 图片内容统一入库。
- 远程图片下载完成前显示骨架和进度。
- 下载后通过受控本地资源协议展示。
- 支持单图、最多四图的紧凑网格和更多图片画廊。
- 支持查看原图、保存副本、在文件管理器中显示、复制提示词信息。
- 加载失败时保留任务信息和重新下载入口。

### 9.5 视频消息

- 异步生成期间显示状态、阶段、已用时间和取消操作。
- 完成后展示封面、时长、尺寸、文件大小和播放按钮。
- 采用原生视频控件播放本地资源。
- 不支持的格式进入“准备播放”状态，通过 FFmpeg 转换为兼容格式。
- 原始文件和转码文件的保留策略应可配置。
- 视频下载失败不能丢失供应商任务 ID，允许重新拉取结果。

### 9.6 工具调用消息

- 默认折叠显示工具名称、所属 MCP、耗时和状态。
- 展开后显示经过脱敏的输入参数和结果摘要。
- 高风险工具调用显示权限说明，而不是只显示“允许/拒绝”。
- 工具执行期间允许取消。
- 工具结果过大时保存为附件，只在消息中展示摘要。

## 十一、核心运行流程

### 11.1 普通文本对话

用户发送消息 → 页面创建待运行消息 → 主进程读取 Agent 快照和历史 → 供应商适配器发起流式请求 → 主进程持续发送文本增量 → 页面合并内容 → 完成后保存最终消息和用量摘要。

约束：

- 页面不持有供应商 API Key。
- 页面不直接访问外部模型地址。
- 流式事件必须包含运行 ID 和消息 ID，防止切换会话后串流。
- 用户停止后主进程必须真正中止网络请求，而不只是停止页面渲染。

### 11.2 Agent 工具调用

模型请求工具 → 运行器校验工具是否可用 → 计算风险等级 → 必要时等待授权 → MCP 执行 → 结果标准化和脱敏 → 写入工具消息 → 将结果继续提交给模型 → 直到得到最终回答或达到循环上限。

约束：

- 默认最大连续工具调用建议为 8 次，可配置但应设硬上限。
- 每个调用拥有独立超时。
- MCP 返回的原始超大内容不得直接注入 DOM。
- 工具调用失败应作为结构化结果交还模型，而不是让整个会话崩溃。

### 11.3 图片生成

Agent 判定图片生成意图 → 调用图片供应商适配器或 MCP 工具 → 创建媒体任务消息 → 接收 Base64、URL 或任务 ID → 必要时轮询 → 主进程下载并校验 → 保存本地文件 → 更新消息内容块 → 页面显示图片。

### 11.4 视频生成

Agent 判定视频生成意图 → 创建异步媒体任务 → 定期查询任务状态 → 更新进度和阶段 → 获得下载地址 → 下载并校验 → 探测格式和时长 → 必要时转码 → 生成封面 → 更新为可播放消息。

### 11.5 应用重启恢复

- 普通文本流：标记为中断，可由用户重试。
- 等待授权的工具调用：标记为中断，不自动执行。
- 正在运行的 stdio 调用：终止旧进程并标记失败。
- 具有供应商任务 ID 的媒体任务：重新查询状态。
- 正在下载的媒体：使用临时文件，支持重新下载，不把半文件标记为完成。

## 十三、技术架构设计

### 13.1 Renderer 层

职责：

- UI 状态和交互。
- 会话列表和消息渲染。
- 流式事件合并。
- 授权对话框和媒体查看器。
- 不接触原始凭据、不创建 MCP 子进程、不直接下载受保护媒体。

### 13.2 Preload/IPC 层

提供明确、窄化的 AI API：

- 配置管理。
- 会话与消息管理。
- 启动、停止和重试运行。
- 工具授权响应。
- 订阅运行增量、工具状态和媒体进度。
- 媒体打开、另存和文件管理器定位。

不得提供“执行任意命令”“请求任意 URL”或“执行任意 SQL”的 AI 专用接口。

### 13.3 主进程 AI 服务层

建议拆分职责：

- AI 配置服务。
- 凭据服务。
- Agent 运行器。
- 模型供应商注册表和适配器。
- MCP 连接管理器。
- MCP 权限策略服务。
- 会话存储服务。
- 媒体任务与下载服务。
- 本地 AI 资源协议服务。
- 运行恢复和诊断服务。

### 13.4 供应商适配器

适配器向上提供统一能力声明：

- 是否支持流式文本。
- 是否支持工具调用。
- 是否支持图片输入。
- 是否支持图片生成。
- 是否支持视频生成。
- 媒体任务是否异步。
- 是否能取消任务。

适配器负责：

- 请求格式转换。
- 响应增量解析。
- 错误映射。
- 任务轮询。
- 媒体结果提取。
- 供应商请求 ID 和用量字段归一化。

### 13.5 MCP 连接管理器

- 每个 LifeOS 用户拥有独立连接集合。
- HTTP 连接按需建立，可复用但必须可关闭。
- stdio 服务按配置启动，限制工作目录和环境变量。
- 应用退出、用户切换、配置禁用时关闭相关连接。
- 工具描述在会话运行前获取和缓存，服务变更后刷新。

### 13.6 媒体服务

- AI 媒体目录必须位于当前用户文件目录下。
- 下载先写临时文件，校验完成后原子重命名。
- 本地协议只能解析受控目录内的相对路径。
- 视频响应支持 Range 请求，确保拖动和大文件播放。
- 媒体 MIME 不可信时通过文件内容检测纠正。
- 图片解码失败或视频探测失败时不进入完成状态。

## 十四、安全与隐私要求

### 14.1 凭据安全

- API Key 不进入 Renderer、日志、错误堆栈和导出文件。
- 表单只显示掩码与末尾少量字符。
- 复制密钥需要明确操作，并沿用自动清空剪贴板策略。
- 备份是否包含凭据必须单独决策，默认建议不包含或使用二次加密。

### 14.2 MCP 权限

- 默认最小权限。
- 读取、写入、删除、命令执行和网络访问应有不同风险等级。
- stdio MCP 的命令配置只能由用户在配置界面保存，模型不得动态改变命令。
- 用户授权只对当前调用生效；“本会话允许”和“永久允许”必须是独立选择。
- 删除、付款、发布、发送消息等外部副作用操作必须逐次确认。

### 14.3 网络与 SSRF

- 外部请求仅由主进程执行。
- 媒体下载默认只接受 HTTPS。
- 阻止访问 file、javascript 等危险协议。
- 默认阻止远程媒体 URL 指向 localhost、回环地址、链路本地地址和内网地址。
- 用户配置的本地模型或本地 MCP 需要单独显式授权，不与媒体下载策略混用。
- 重定向后必须再次验证目标地址。

### 14.4 内容渲染

- Markdown 经 DOMPurify 清理。
- 禁止脚本、iframe、object、embed 和内联事件。
- 外部链接通过受控 `openExternal` 打开并校验协议。
- 工具结果不能作为可信 HTML 注入。
- 代码块只展示，不自动执行。

### 14.5 文件安全

- 文件名由应用生成或严格清理。
- 不接受路径穿越、绝对路径注入和符号链接逃逸。
- 对媒体大小、类型、下载时长和磁盘剩余空间设限。
- 删除操作仅作用于 AI 媒体目录中的已登记文件。

## 十五、错误与状态设计

| 场景 | 用户状态 | 建议操作 |
| --- | --- | --- |
| 未配置供应商 | 引导态 | 创建供应商或导入配置 |
| API Key 无效 | 配置错误 | 编辑凭据并重新测试 |
| 模型不存在 | 模型错误 | 检查模型名称或刷新能力 |
| 供应商限流 | 等待/失败 | 显示可重试时间，避免立即循环重试 |
| 网络中断 | 中断 | 保留已生成内容并重试 |
| MCP 启动失败 | 工具不可用 | 查看命令、路径和 stderr 摘要 |
| MCP 调用超时 | 工具失败 | 取消、重试或禁用该工具 |
| 用户拒绝授权 | 已拒绝 | 将拒绝结果交给 Agent继续回答 |
| 媒体生成排队 | 生成中 | 显示阶段、时间和取消入口 |
| 临时 URL 过期 | 下载失败 | 使用任务 ID 重新获取结果 |
| 视频格式不支持 | 准备播放 | 转码或保留原文件 |
| 磁盘空间不足 | 存储错误 | 清理缓存或更换目录 |
| 应用重启 | 已中断/恢复中 | 恢复媒体任务，文本任务可重试 |

错误信息采用“发生了什么 + 可能原因 + 下一步”的结构，默认隐藏原始堆栈和敏感响应。

## 十六、非功能要求

### 16.1 性能

- 不考虑供应商网络耗时，应用收到首个增量后 100ms 内完成界面更新。
- 流式文本更新应批量合并，避免逐 token 触发昂贵重排。
- 1000 条消息的会话仍可打开和滚动，长会话需要虚拟化或分页。
- 图片使用缩略图或受控尺寸解码，避免原图一次性占用过多内存。
- 视频下载和转码不得阻塞主窗口交互。

### 16.2 稳定性

- 每个运行使用独立 AbortController 或等效取消机制。
- MCP 子进程崩溃不得导致 Electron 主进程退出。
- 媒体下载使用临时文件和原子完成状态。
- 数据库写入失败时不向页面报告成功。
- 同一会话默认只允许一个前台 Agent 运行，媒体后台任务可独立继续。

### 16.3 兼容性

- macOS 为首要验收平台。
- Windows 需要验证 stdio 命令、路径引号、环境变量和进程关闭。
- 支持 LifeOS 现有四套主题和中英文。
- 应用最小窗口 800×600 下核心发送、停止和播放操作必须可用。

### 16.4 可维护性

- 供应商适配器、MCP 管理、媒体服务和 UI 消息块相互解耦。
- 新供应商接入不应要求修改通用消息渲染器。
- 数据库 schema 具有版本与迁移测试。
- 所有 IPC 输入经过运行时校验，不能只依赖 TypeScript 类型。

### 16.5 可观测性

- 默认不发送外部遥测。
- 本地保存有限的运行诊断：时间、阶段、错误码、供应商请求 ID、MCP 名称和耗时。
- 日志对密钥、认证头、用户隐私文本和工具敏感参数进行脱敏。
- 用户可以复制脱敏诊断用于排障。

## 十七、验收标准

### 17.1 配置验收

- 可创建一个文本供应商、一个 Agent 和一个 MCP 服务。
- 连接测试能准确区分成功与主要失败类型。
- 重启应用后配置仍存在，敏感值不以明文出现在普通数据库和页面状态中。
- 删除有依赖的配置时不会产生悬空 Agent。

### 17.2 文本对话验收

- 连续完成不少于 10 轮多轮对话。
- 文本在响应过程中逐步显示。
- 停止生成后网络请求终止，部分回答被保留并明确标记。
- 切换会话时不会把旧会话增量写入新会话。
- 重启应用后历史消息顺序和内容正确。

### 17.3 MCP 验收

- HTTP 和 stdio 各连接一个真实 MCP 服务。
- 成功发现工具并完成一次只读调用。
- 完成一次需要确认的高风险模拟调用。
- 拒绝授权后 Agent 能继续给出非工具回答。
- MCP 超时、进程退出和非法结果均有明确状态，不造成应用崩溃。

### 17.4 图片验收

- 分别验证 Base64 图片、远程 URL 图片和 MCP 图片内容。
- 图片完成后在对话内显示，重启后仍可打开。
- 远程 URL 失效后本地图片仍可访问。
- 加载失败、磁盘不足和错误 MIME 有可恢复提示。

### 17.5 视频验收

- 完成一次真实异步视频生成任务。
- 生成期间显示状态，完成后可在对话中播放。
- 重启后视频仍可播放。
- 支持取消、下载失败重试和临时 URL 重新获取。
- 对不兼容格式至少能给出转码状态或明确诊断。

### 17.6 安全验收

- Renderer 无法读取完整 API Key。
- 任意 MCP 不能绕过授权执行高风险工具。
- 媒体 URL 无法通过重定向访问受阻止的本地/内网地址。
- 本地资源协议无法读取 AI 媒体目录外的文件。
- Markdown 和工具结果无法执行脚本。
- 日志和导出内容不包含认证头和完整密钥。

## 十八、测试策略

### 18.1 单元测试

- 供应商响应标准化。
- 流式增量合并与终态判定。
- 工具风险分类和授权策略。
- MCP 配置校验与错误映射。
- 媒体 MIME、文件名、路径和 URL 安全校验。
- 视频任务状态迁移。
- 会话删除与媒体引用处理。

### 18.2 集成测试

- Renderer → preload → 主进程 → 模拟供应商的完整流式链路。
- Agent → MCP → 工具结果 → Agent 最终回答。
- 媒体任务 → 下载 → 数据库 → 本地协议 → 页面展示。
- 应用重启后的任务恢复。
- 用户切换时的连接和数据隔离。

### 18.3 打包应用人工测试

- macOS 安装包内访问远程模型和本地 MCP。
- Windows stdio 路径与子进程关闭。
- 无网络、代理、证书失败、磁盘不足和文件权限不足。
- 中文、英文、四套主题、800×600 和高 DPI。
- 键盘导航、屏幕阅读器基本语义和减少动态效果。

## 二十一、主要风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 不同供应商媒体协议差异大 | 适配成本和错误率上升 | 供应商适配器注册表，首版限定真实验收供应商 |
| 视频任务耗时长、链接易过期 | 重启后结果丢失 | 保存任务 ID，完成后立即下载，本地持久化 |
| MCP 可执行高风险操作 | 数据或外部系统受损 | 最小权限、逐次确认、工具白名单和审计记录 |
| stdio 跨平台差异 | Windows/macOS 行为不一致 | 结构化命令配置、真实打包验收和进程生命周期测试 |
| API Key 泄露 | 严重安全问题 | 主进程凭据服务、Renderer 隔离、日志脱敏 |
| 长会话和大媒体占用资源 | 卡顿、磁盘爆满 | 虚拟列表、缩略图、容量上限、清理和磁盘预检 |
| 供应商内容安全策略不同 | 请求失败或结果不可用 | 显示供应商原始错误的脱敏摘要，不承诺绕过策略 |
| AI 扩展影响当前 RC 稳定性 | 发布风险 | 独立分支、独立数据库、模块级故障隔离和分阶段门禁 |
