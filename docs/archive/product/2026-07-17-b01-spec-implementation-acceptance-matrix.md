## 1. 使用规则

状态定义：

- **已完成**：规格要求已有实现，且有代码或自动化测试证据；仍需在 D 阶段完成打包应用人工验收的，不影响这里的代码级状态。
- **部分完成**：主链路存在，但范围、体验、持久化、异常路径或规格承诺尚未完全覆盖。
- **未开始**：当前代码中没有可作为 v1.0 交付的实现证据。
- **延期**：已明确移出当前 RC 范围，进入后续路线图。

本矩阵将“代码已存在”和“可对外承诺”分开记录。标记为“部分完成”的能力不得在 RC 发布说明中描述为完整能力。

## 11. RC 范围基线

### 允许作为当前 RC 核心承诺

- 本地账户、任务基础视图与任务数据管理。
- Markdown 笔记、笔记本组织和基础导出。
- EPUB/PDF 为主的书库与阅读器能力。
- 已验证的 YouTube / Bilibili 视频解析、下载和本地播放链路。
- 真实加密密码库、旧数据迁移和安全测试。
- 手动用户数据备份、校验和恢复最小闭环。
- 中文 / English、主题、视频设置、系统更新入口（真实升级仍未通过前不得宣传为完成）。

### 必须降级描述或标记为演示能力

- 任务自然语言解析。
- 汇率数据。
- Dashboard 固定小工具、卡片重排。
- 视频平台的未验证扩展范围。
- 全局统一标签和跨模块搜索。

### 明确移出当前 RC

- 云同步、移动端、AI、插件市场、网页剪藏、日历集成。
- 系统托盘任务数。
- 笔记多编辑器、完整版本历史、30 天回收站。
- 自动备份保留策略、增量备份和全目录迁移。
- Windows / Ubuntu 发布级验收。

## 12. 结论与后续任务

B-01 已完成：每项主要 v1.0 规格均有状态、实现证据和验收口径。当前产品不应继续按“全功能个人操作系统”扩大承诺，而应以本矩阵为范围基线推进：

1. B-02：根据本矩阵将部分完成/未开始能力从当前 RC 文案和规格中降级或延期。
2. B-03：清理或标记任务模板、调度日志、汇率和更新模拟数据。
3. B-04：决定全局搜索是补齐视频，还是正式收窄产品文案。
4. B-05：对任务、笔记和 Dashboard 的关键差异建立 P0/P1 验收结论。
5. C-01 至 C-05：完成真实发布、签名和升级演练后，才能提高发布准备度。
