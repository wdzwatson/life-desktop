# B-04 全局搜索范围决策

日期：2026-07-17
任务：B-04
状态：已完成
决策：补齐视频搜索；工具名称暂不纳入全局搜索

## 1. 原问题

设计规格要求全局搜索覆盖任务标题、笔记全文、书名作者、视频标题和工具名称。原实现只查询任务、笔记和书籍，顶部搜索框文案却同时提到视频，导致 UI 承诺与实际结果不一致。

## 2. 处理方案

本轮选择补齐视频搜索，保持产品规格的核心四类内容范围：

- 任务：标题、描述。
- 笔记：标题、正文。
- 书籍：书名、作者。
- 视频：标题、普通 URL、规范来源 URL。

工具名称暂不加入全局搜索，因为工具箱当前是静态面板入口，没有统一可检索实体、持久化名称和结果跳转模型。工具箱继续通过侧边栏和快捷入口访问，不扩大搜索承诺。

## 3. 实现调整

- `src/App.tsx`
  - 在命令面板查询中增加 `videos.db` 查询。
  - 视频结果最多返回 3 条，查询 `title`、`url` 和 `source_url`。
  - 点击结果进入视频模块并关闭搜索面板。
- `src/locales/zh-CN.json`、`src/locales/en-US.json`
  - 增加视频结果的来源、时长和未知值文案。
- 保持原有 `/task` 和 `/note` 快捷指令行为不变。

## 4. 验收标准

- 搜索包含视频标题时能返回视频结果。
- 搜索视频 URL 或来源 URL 片段时能返回视频结果。
- 视频结果显示来源和时长，缺失字段有明确占位文案。
- 点击视频结果进入视频页，不影响任务、笔记和书籍结果的跳转。
- 不对工具箱功能做不具备实体基础的搜索承诺。
- 数据查询继续使用参数绑定，不拼接用户输入到 SQL 语句。

## 5. 验收证据

- `npm test`：238 + 20 + 11 + 10 项全部通过。
- `npm run build`：TypeScript、Vite、Electron 主进程和 preload 构建通过。
- `npm run lint`：通过。
- `git diff --check`：通过。
- 现有 SQL 绑定安全测试继续通过。

## 6. 后续事项

- 若未来工具箱采用可注册工具实体模型，可单独增加工具搜索；不在当前 RC 范围内。
- 视频结果进入详情或定位具体记录的能力可在 D 阶段根据用户旅程继续增强；当前先保证结果范围和入口一致。

## 7. 结论

B-04 已完成。全局搜索的 UI 文案和实际查询范围现在一致覆盖任务、笔记、书籍和视频；工具箱搜索被明确排除，避免继续产生“规格写了但没有实现证据”的产品承诺。
