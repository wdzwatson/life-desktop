# B-03 演示数据清理与标记

日期：2026-07-17
任务：B-03
状态：已完成
依据：B-01 规格—实现—验收矩阵、B-02 RC 范围基线

## 1. 问题盘点

本任务处理四类容易被误认为真实业务能力的数据或状态：

| 类别 | 原问题 | 处理结果 |
|---|---|---|
| 任务定时日志 | 页面固定填充“每日备份、每分钟扫描、每周归档”等静态记录；“立即执行”只显示 toast | 改为读取当前用户真实 `recurring_rules`；无规则显示真实空状态；操作按钮改为查看规则 |
| 任务模板 | 内置模板是产品功能，不是用户历史数据 | 保留模板入口；继续以“模板”命名，不混入任务历史和定时运行记录 |
| 汇率工具 | 使用固定汇率常量，却显示为离线缓存汇率 | 将入口和提示改为“示例汇率 / 非实时数据”，避免产生实时数据误解 |
| 自动更新 | 开发环境检查、下载、安装为 mock 流程 | 开发 mock 返回状态后，在设置页显示明确提示；生产环境仍使用 electron-updater 分支 |

## 2. 实现调整

- `src/views/Tasks.tsx`
  - 删除静态 `scheduledLogs` 数据数组。
  - 从数据库加载的 `rules` 派生周期规则概览。
  - 删除伪造“立即执行”行为，改为跳转查看实际规则。
  - 增加无周期规则空状态和说明。
- `src/views/Settings.tsx`
  - 读取检查/下载更新 IPC 的 `isMock` 返回值。
  - 开发模拟更新时展示开发环境提示。
- `src/locales/zh-CN.json`、`src/locales/en-US.json`
  - 增加真实周期规则、空状态和开发模拟提示文案。
  - 将汇率文案改为示例数据语义。
  - 保留 `btn_run_now` 兼容文案键，但其显示语义改为“查看规则”，不再触发伪运行。

## 3. 验收标准

- 定时页不再默认出现不存在的备份、扫描、归档运行记录。
- 当前用户没有周期规则时，页面明确显示空状态。
- 已保存的周期规则能在概览中显示，并可返回真实规则编辑页。
- 汇率模块不再暗示数据来自实时网络或最近缓存。
- 开发环境更新流程明确显示为模拟，不影响生产环境更新分支。
- 中英文文案键完整，既有 recurring rule locale 测试继续通过。
