## 4. 核心试用脚本

每位试用人员建议按以下顺序执行，耗时约 45-75 分钟。

### 4.1 启动与账户

- 首次启动 LifeOS。
- 使用 guest 进入。
- 新建一个本地账户，设置昵称、头像和密码。
- 退出登录，再用该账户进入。
- 重启应用，确认账户仍可进入。

记录重点：

- 是否理解 guest 与本地账户的差异。
- 密码错误提示是否清晰。
- 首次启动是否有卡顿、闪退或空白页。

### 4.2 任务

- 创建 3 个任务，其中 1 个设置截止日期。
- 为一个任务创建至少 2 层子任务。
- 在看板中移动任务状态。
- 切到日历查看截止日期。
- 创建一个周期任务规则。
- 重启应用后确认任务仍存在。

记录重点：

- 创建入口是否明显。
- 子任务层级是否容易理解。
- 看板拖动是否稳定。

### 4.3 笔记

- 新建 2 篇笔记。
- 使用 Markdown 标题、列表、链接。
- 创建或使用分类。
- 尝试双链引用。
- 搜索笔记。
- 导出或复制内容。

记录重点：

- 编辑与预览是否符合预期。
- 搜索结果是否容易定位。
- 恶意 HTML 或脚本内容是否被阻止。

### 4.4 书库与阅读器

- 导入 1 本 EPUB 或 PDF。
- 打开阅读器。
- 滚动或跳章。
- 添加批注。
- 导出批注。
- 删除测试书籍。

记录重点：

- 导入失败提示是否可理解。
- PDF / EPUB 阅读体验是否有截断。
- 批注导出是否可打开。

### 4.5 视频

- 检查视频工具状态。
- 添加一个公开视频 URL。
- 若本机有 `yt-dlp` / `ffmpeg`，尝试解析与下载。
- 若工具缺失，确认提示是否可理解。
- 添加分组与标签。
- 导入或播放本地视频文件。

记录重点：

- 工具缺失时是否知道下一步怎么做。
- 下载失败原因是否保留。
- 本地播放是否稳定。

### 4.6 设置、语言、主题与异常

- 切换 English，再切回中文。
- 循环四种主题。
- 缩小窗口使用 3 分钟。
- 打开全局搜索，按 Escape 关闭。
- 查看数据与备份说明。
- 查看系统更新页。

记录重点：

- 是否存在不可读文字、遮罩无法关闭、按钮无响应。
- 更新失败或工具缺失提示是否会误导。
- 备份/恢复说明是否让人有安全感。

## 6. 严重等级判定规则

| 等级 | 判定标准 | 示例 | 默认处理 |
| --- | --- | --- | --- |
| P0 | 数据丢失、无法启动、核心数据库损坏、恢复破坏原数据、阻塞全部使用 | 打开即崩溃；恢复失败后原数据消失 | 立即阻断 RC，必须修复或回退 |
| P1 | 核心流程阻塞，但不造成数据丢失，有绕行成本 | 任务无法保存；书籍无法打开；视频失败无可读原因 | RC 前优先修复或明确延期批准 |
| P2 | 局部体验缺陷，可绕过，不阻塞核心流程 | 文案不清晰；窄窗口局部排版不佳 | 进入修复队列，按风险排序 |
| 建议 | 非缺陷，属于功能增强或偏好 | 希望支持更多主题；希望自动标签 | 进入产品 backlog |
