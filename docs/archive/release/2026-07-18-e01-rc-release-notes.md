## 1. RC 定位

LifeOS v1.0.2 当前定位为内部 RC 候选版，用于小范围真实使用反馈和最终发布决策。

本版本不承诺公开分发能力，原因是 macOS Developer ID 签名、公证、GitHub published release 与真实自动更新演练仍未解除阻塞。内部试用应把重点放在本地数据、核心工作流、异常反馈和用户理解成本上。

## 4. 安装与启动说明

### 4.1 macOS 内部试用

1. 获取 `LifeOS-1.0.2.dmg` 或 `LifeOS-1.0.2-mac.zip`。
2. 校验 SHA256 是否与本说明一致。
3. 启动前备份现有数据目录，见“数据目录与备份建议”。
4. 打开 LifeOS。

注意：当前包未完成 Developer ID 签名与公证。macOS Gatekeeper 可能提示未验证开发者或包已损坏。该情况是已知发布阻塞，不应向外部用户分发未签名包。

### 4.2 不在本轮 RC 范围

- Windows / Ubuntu 安装包与平台专属行为。
- 正式签名、公证和外部公开下载页。
- 真实自动更新从旧版本升级到新版本。

## 5. 数据目录与备份建议

默认数据根目录：

```text
/Users/mac/LifeOS
```

建议内部试用前执行：

```bash
cp -R /Users/mac/LifeOS /Users/mac/LifeOS.backup-before-rc-20260718
```

应用内也提供“设置 -> 数据与备份”的备份与恢复功能。备份包会包含当前用户的必要配置、数据库、笔记、书籍和默认视频目录文件，并排除敏感 vault legacy backup。外部视频目录只记录在 manifest 中，不会自动复制。

恢复功能会先校验 manifest、路径和 checksum，再替换当前用户数据；恢复失败应回滚并保留敏感恢复备份。

## 6. 已知问题与限制

| 等级 | 项目 | 状态与说明 |
| --- | --- | --- |
| P0 | macOS 签名与公证 | 当前机器无有效 Developer ID Application identity，`electron-builder` 跳过签名。正式发布前必须完成签名、公证、Gatekeeper 验证。 |
| P0 | 真实自动更新演练 | 当前无 GitHub published release / release metadata，C-05 真实升级与失败恢复演练未通过。不得宣传自动更新已可用于生产。 |
| P1 | 视频真实下载成功路径 | 当前干净测试环境缺少 `yt-dlp` 和 `ffmpeg`，D-06/D-08 验证了缺工具提示、失败、重试和本地播放；真实下载成功与 Cookie 授权需安装工具后补测。 |
| P1 | 磁盘不足 | D-08 使用不可写/非法目标路径覆盖写入失败；未在系统盘制造真实满盘场景。正式发布前建议在沙箱卷补测 ENOSPC。 |
| P1 | 内部更新 UI | 失败反馈可读，但真实下载、安装、回滚仍依赖 C-05。 |
| P2 | 高 DPI | 使用 Electron `deviceScaleFactor: 2` 和截图近似验收；建议在目标 Retina 设备上人工复核。 |
| P2 | 外部视频源稳定性 | 平台策略、登录态、Cookie 和会员权限可能影响下载结果；失败诊断与重试已存在，但成功率依赖外部服务。 |

## 7. 不应夸大的能力

- 不把开发环境 mock 更新流程描述为生产自动更新。
- 不宣称未签名包适合公开分发。
- 不宣称视频平台下载在所有 URL、所有清晰度和所有登录态下稳定成功。
- 不宣称 Windows / Ubuntu 已完成 RC 验收。
- 不把磁盘满、安装失败回滚等未完成真实环境演练的场景描述为已完全验证。
