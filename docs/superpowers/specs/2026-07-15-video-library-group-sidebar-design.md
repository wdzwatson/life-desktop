# 视频管理分组栏 UI 与交互设计

日期：2026-07-15  
状态：待用户审阅

## 1. 背景

当前视频管理页左侧栏把“全部”“未分类”、用户分组和标签放在同一个卡片中。固定入口与可管理分组外观相同，但每个自定义分组又长期显示编辑、删除按钮，导致行内拥挤、名称空间不足，也无法清楚表达父子层级。

视频分组与书库分类存在两个关键差异：

- 视频分组支持父子层级，选择父分组时会同时筛选其后代分组中的视频。
- 视频分组名称需要支持当前及未来新增语言的独立译名，不能在组件中写死中文和英文。

本设计沿用书库侧栏已经确认的“固定入口 + 用户内容 + 右键管理”方向，同时保留视频分组的层级能力。

## 2. 设计目标

1. 明确区分系统固定入口与用户创建的分组。
2. 使用可折叠树形结构清晰呈现父子分组。
3. 移除每行常驻的编辑、删除按钮，使默认界面稳定、简洁。
4. 保留一级分组、子分组的新建、重命名、多语言编辑和删除能力。
5. 删除分组时保留视频与子分组，明确展示影响范围。
6. 使用数据库事务保证层级、翻译和视频归属同步更新。
7. 从应用已注册语言动态生成翻译编辑项，新增语言时无需修改视频或书库组件。

## 3. 信息架构

左侧栏分为三个区域。

### 3.1 快速访问

固定显示在顶部，不提供管理菜单：

- **全部视频**：展示所有视频。
- **待整理**：替代侧栏中的“未分类”，展示 `group_id` 为空或指向不存在分组的视频。

两个入口使用与书库侧栏一致的行高、选中态、图标和数量布局。“待整理”数量大于零时使用克制的暖色提示；底层 `NULL` 分组值不改变。

### 3.2 我的分组

- 标题为“我的分组”，右侧唯一的 `＋` 只创建一级分组。
- 自定义分组使用可折叠树展示。
- 父分组左侧显示展开箭头；无子分组时保留对齐空间但不显示箭头。
- 左键分组名称筛选该分组及所有后代分组中的视频。
- 点击展开箭头只展开或收起节点，不改变当前筛选。
- 右键打开管理菜单，不改变当前筛选。
- 每行显示分组图标、当前语言名称和直属视频数量。父分组被选中后，右侧结果仍包含后代视频。

### 3.3 标签

标签继续作为独立区域位于分组树下方，保持现有筛选逻辑和标签胶囊样式。本次不增加标签重命名、删除或多语言管理。

## 4. 树形展示与折叠状态

- 一级分组默认展开。
- 新建子分组成功后自动展开父分组，并选中新子分组。
- 选择深层分组时自动展开它的全部祖先。
- 用户主动折叠的状态仅保存在当前视频页面会话中，不写入数据库。
- 切换应用语言不改变展开状态和当前选中分组。
- 树形构建必须防御循环父子关系和失效 `parent_id`：循环节点或孤立节点作为一级节点展示，保证侧栏仍可操作。
- 缩进深度在视觉上设置合理上限，但数据和筛选逻辑不限制实际层数。

## 5. 管理交互

### 5.1 新建一级分组

点击“我的分组”右侧 `＋`：

1. 在分组树顶部显示行内输入框。
2. Enter 保存，Esc 或明确的取消按钮取消。
3. 空名称或当前父级下的重复名称不能提交；错误时保留输入内容。
4. 创建成功后自动选中新分组。
5. 规范名称和当前应用语言的译名同时写入数据库。

### 5.2 右键菜单

只有自定义分组响应右键。菜单项目按以下顺序显示：

1. **新增子分组**：在目标分组的子节点区域显示行内输入框，并展开目标分组。
2. **重命名**：目标行切换为当前语言名称的行内编辑状态。
3. **编辑其他语言名称…**：打开多语言名称对话框。
4. 分隔线。
5. **删除分组…**：危险操作，打开确认对话框。

菜单通过选择操作、点击外部、按 Esc、切换页面或目标分组消失而关闭。菜单打开后焦点移入第一项，关闭后恢复到原分组行。

### 5.3 新建子分组

- “新增子分组”只出现在父分组右键菜单中，不在行内常驻显示按钮。
- 输入框显示在父分组之后、现有子节点之前，并使用下一级缩进。
- Enter 保存，Esc 取消。
- 保存期间阻止重复提交。
- 创建成功后保持父分组展开并选中新子分组。

### 5.4 重命名与多语言名称

- 行内重命名默认选中当前语言下显示的完整名称。
- Enter 保存，Esc 取消；保存失败时保留输入和错误提示。
- 重命名同时更新分组规范名称和当前语言译名，其他语言译名保持不变。
- “编辑其他语言名称…”对话框列出所有已注册语言，而不是固定列出中文和英文。
- 当前语言输入放在首位，其余语言可在“更多语言”区域中编辑。
- 某个语言译名为空时删除该语言的翻译记录，显示时回退到规范名称。
- 同一父级下，同一种语言的最终显示名称不能重复；不同父级可以使用相同名称。

### 5.5 删除分组

删除确认对话框显示：

- 分组当前语言名称。
- 直属视频数量。
- 直属子分组数量。
- 明确说明视频不会被删除，直属视频将进入“待整理”，子分组将提升到上一级。

确认后在单个事务中：

1. 将直属视频的 `group_id` 更新为 `NULL`。
2. 将直属子分组的 `parent_id` 更新为被删分组的父级；删除一级分组时，子分组提升为一级。
3. 删除该分组的全部翻译。
4. 删除分组记录。

删除完成后：

- 如果删除的是当前选中分组，筛选切换到“全部视频”。
- 如果详情抽屉或批量编辑草稿引用该分组，将其分组值更新为无分组。
- 其他筛选、标签选择和树形折叠状态保持不变。

任一步骤失败时整个事务回滚，对话框保持打开并显示错误，不展示虚假的成功状态。

## 6. 动态多语言设计

### 6.1 语言注册

- 应用提供共享的语言资源注册模块，集中生成 i18next `resources` 和可配置语言列表。
- 语言资源从 `src/locales/*.json` 动态发现，语言代码取自文件名，例如 `ja-JP.json` 注册为 `ja-JP`。
- 视频、书库和设置页均从共享注册模块读取语言列表，不再维护各自的 `SUPPORTED_LOCALES` 常量。
- 语言显示名称通过 `Intl.DisplayNames` 按当前界面语言生成；运行环境不支持或无法识别时回退为语言代码。
- 新增语言时，加入对应 JSON 资源文件即可出现在设置页语言选择和分组翻译编辑器中。

### 6.2 分组翻译数据

视频数据库新增 `video_group_translations` 表：

```sql
CREATE TABLE IF NOT EXISTS video_group_translations (
  group_id INTEGER NOT NULL,
  locale TEXT NOT NULL,
  translation TEXT NOT NULL,
  PRIMARY KEY (group_id, locale),
  FOREIGN KEY (group_id) REFERENCES video_groups(id) ON DELETE CASCADE
);
```

显示名称的回退顺序为：

1. 当前 i18n 语言的非空译名。
2. `video_groups.name` 规范名称。

视频与分组之间继续通过稳定的 `group_id` 关联，切换语言或修改译名不改变视频归属。

### 6.3 同级唯一性

- 规范名称在同一父级内不允许重复，比较前去除首尾空格并进行不区分大小写的比较。
- 每个语言的最终显示名称也在同一父级内保持唯一。
- SQLite 现有 `video_groups.name UNIQUE` 是全局限制，需要迁移为基于 `COALESCE(parent_id, -1)` 与规范名称的同级唯一索引。
- 翻译唯一性涉及父级联表关系，由事务写入前查询验证；验证失败不写入任何名称或翻译。

## 7. 组件与模块边界

实施时拆分以下职责：

- `VideoGroupSidebar`：负责快速访问、分组树、一级新增入口和标签区域布局。
- `VideoGroupTreeRow`：负责节点缩进、展开箭头、选择状态和行内编辑。
- `VideoGroupContextMenu`：负责右键菜单定位、键盘导航与焦点恢复。
- `AccessibleDialog`：复用书库页已有实现，承载多语言编辑和删除确认。
- `videoGroupSidebarUtils`：负责树构建、祖先/后代计算、折叠可见节点、名称校验、计数和删除后的界面状态。
- 共享语言注册模块：负责动态发现语言资源、生成 i18next resources 和语言显示信息。

`Videos.tsx` 继续持有分组、翻译、视频、标签和当前筛选状态，并向侧栏组件传入数据与事务操作回调。本次不重构视频解析、下载、播放或详情抽屉的主体结构。

## 8. 数据流与事务

### 8.1 加载

视频页并行加载分组、分组翻译、标签和视频。加载后：

1. 生成当前语言的分组显示名称。
2. 构建防循环的树形模型。
3. 计算全部视频、待整理和各分组直属视频数量。
4. 根据当前选中分组计算包含后代的筛选集合。

切换语言时只重新计算显示名称和排序，不重新写入数据。

### 8.2 写入

新建、重命名、多语言编辑和删除都通过现有 `dbTransaction('videos', statements)` IPC 桥执行。每次事务成功后重新加载分组、翻译和视频；事务失败时重新读取真实数据，但保留仍有意义的输入内容和错误信息。

### 8.3 兼容迁移

- 现有分组的 `name` 直接作为规范名称保留，ID、父子关系和视频归属不变。
- 不为历史分组自动生成所有语言译名；没有译名时使用规范名称。
- 重建分组表或索引时必须保留原有 ID、`parent_id`、排序和时间字段。
- 迁移完成后验证没有丢失分组、视频引用和父子关系。

## 9. 视觉规则

- 侧栏宽度与书库侧栏保持接近，分组名称区域可伸缩，数量区域固定在右侧。
- 默认、悬停、选中和右键菜单打开状态不改变行内布局。
- 层级主要通过缩进和展开箭头表达，不为每层使用高饱和背景色。
- 长名称使用单行省略，完整路径通过 `title` 或可访问描述提供。
- 右键菜单和对话框沿用书库页视觉语言。
- 标签区域与分组树之间使用标题和留白区隔，不使用过重分割线。
- 所有颜色使用主题变量，并在现有主题下保持可读对比度。

## 10. 可访问性与桌面行为

- 固定入口和分组行均可通过键盘聚焦并按 Enter 选择。
- 展开箭头提供 `aria-expanded` 和明确的可访问名称。
- 右键菜单使用 `role="menu"` / `role="menuitem"`，支持上下方向键循环、Enter 执行、Esc 关闭。
- 行内输入框提供与操作和父级相关的可访问名称。
- 多语言和删除对话框锁定焦点，支持 Esc 关闭，并在关闭后恢复到来源分组行。
- 右键不会触发左键筛选；展开箭头不会触发分组选择。
- 菜单靠近窗口边缘时自动调整方向，不能超出可视区域。

## 11. 错误处理

- 空名称、同级规范名称重复或同语言显示名称重复时，在输入区域显示具体错误。
- 保存过程中禁用重复提交，但不冻结侧栏的普通筛选操作。
- 数据库事务失败时保留编辑上下文并显示可本地化错误。
- 删除目标在确认前已不存在时，关闭对话框、重新加载数据并提示分组不可用。
- 当前选中分组在外部数据变化后不存在时，安全回退到“全部视频”。
- 动态语言资源缺少某个系统文案键时沿用 i18next fallback language；分组译名缺失时回退到规范名称。

## 12. 测试与验收

### 12.1 工具函数与数据测试

- 正确构建多层树、孤立节点和循环关系的安全结果。
- 展开、收起和祖先自动展开逻辑正确。
- 父分组筛选包含全部后代视频。
- “待整理”包含空分组和失效分组引用。
- 同级名称重复被拒绝，不同父级同名允许。
- 删除后的选中状态、视频归属和子分组父级正确。
- 动态注册第三种测试语言后，语言列表和显示名称自动出现。
- 当前语言译名、规范名称回退和空译名删除行为正确。

### 12.2 组件交互测试

- `＋` 只创建一级分组。
- 只有右键菜单提供“新增子分组”。
- 左键、展开箭头和右键三种操作互不误触发。
- 行内新建、重命名支持 Enter、Esc、重复校验和失败保留。
- 右键菜单键盘导航、焦点恢复和边缘定位正确。
- 删除对话框显示直属视频数和直属子分组数。
- 删除事务成功后直属视频进入“待整理”，子分组提升。
- 多语言编辑器动态显示全部已注册语言，并正确保存或删除译名。

### 12.3 集成与视觉验收

- 视频、分组、标签、下载和详情现有流程无回归。
- 书库翻译编辑器和设置页语言选择改用共享语言列表后行为不变。
- 新增一个测试语言资源后，无需修改视频、书库或设置组件即可显示该语言。
- 长名称、深层缩进、小窗口、浅色和深色主题下布局稳定。
- 完整测试、lint 和生产构建通过，并在 Electron 页面中人工验证右键、折叠、翻译和删除流程。

## 13. 非目标

- 不实现拖拽排序、拖拽改父级或批量移动分组。
- 不实现标签的层级、多语言或管理菜单。
- 不持久化树的展开状态。
- 不改变父分组包含后代视频的现有筛选语义。
- 不重构视频下载、解析、播放和详情编辑主体。
- 不在本次工作中清理应用其他区域全部历史硬编码双语文案；本次新增及受影响界面必须使用共享语言注册和 i18n key。
