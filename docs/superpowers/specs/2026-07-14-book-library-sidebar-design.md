# 书库分类栏 UI 与交互设计

日期：2026-07-14  
状态：待用户审阅

## 1. 背景

当前书库左侧分类栏把“全部书架”“未分类”和用户创建的分类放在同一列表中。三类入口看起来相同，但行为不同：

- “全部书架”和“未分类”是系统固定入口，不能编辑或删除。
- 自定义分类可以编辑和删除，悬停时显示操作按钮。
- 操作按钮出现后会占用行内空间，使名称、数量和操作区的布局发生变化。

这让固定入口看起来像两个受限制的自定义分类，也让管理操作显得拥挤且不稳定。

## 2. 设计目标

1. 明确区分系统固定入口与用户创建的书架。
2. 让左侧栏在默认、悬停和选中状态下保持简洁、稳定。
3. 保留自定义书架的新增、重命名、翻译编辑和删除能力。
4. 删除书架前明确说明影响，避免用户误以为书籍也会被删除。
5. 保持现有分类数据和书籍归属逻辑兼容，不修改数据库结构。

## 3. 信息架构

分类栏分为两个区域。

### 3.1 快速访问

固定显示在顶部，不提供管理菜单：

- **全部书籍**：展示书库中的所有书籍。中文名称从“全部书架”改为“全部书籍”，使名称与实际筛选行为一致。
- **待整理**：替代界面文案“未分类”，展示没有有效自定义分类的书籍。它是收件箱或待处理状态，不是正式书架。

“待整理”在数量大于零时使用克制的暖色提示；数量为零时恢复普通次要文字颜色。底层数据仍兼容 `未分类`、`Uncategorized`、空值及失效分类等现有情况。

### 3.2 我的书架

只显示用户创建的自定义分类。

- 分组标题为“我的书架”。
- 标题右侧显示唯一的新增按钮 `＋`。
- 每行显示书架图标、名称和书籍数量。
- 左键选择书架并筛选右侧书籍。
- 右键打开书架管理菜单。
- 普通悬停只改变行背景和文字颜色，不显示编辑、删除图标，也不改变行内布局。

## 4. 视觉规则

### 4.1 行布局

- 分类栏宽度保持当前约 240px，不因操作状态改变宽度。
- 每行高度约 36–40px，图标、名称和数量垂直居中。
- 数量使用固定的右侧区域或胶囊样式，避免名称长度变化时左右跳动。
- “快速访问”与“我的书架”之间通过分组标题和留白建立层级，不使用过重的分割线。

### 4.2 状态

- **默认**：中性文字与透明背景。
- **悬停**：轻微背景色；不出现操作图标。
- **选中**：使用主题强调色的浅色背景与强调文字。
- **右键菜单打开**：对应行保持悬停背景，菜单浮于行右侧。
- **待整理有内容**：使用暖色文字或数量胶囊，但不使用高饱和警告红色。

所有颜色继续使用现有 CSS 主题变量，并为不同主题补充必要的半透明状态色。

## 5. 管理交互

### 5.1 新增书架

点击“我的书架”右侧 `＋`：

1. 在自定义书架列表顶部插入一个行内输入框。
2. 输入名称后按 Enter 创建。
3. 按 Esc 或点击侧栏外部取消。
4. 空名称不能提交；名称重复时保留输入状态并显示轻量错误提示。
5. 创建成功后自动选中新书架。

新增过程中继续保留现有多语言数据能力：新书架先写入当前语言名称，其他语言名称可在管理菜单中继续编辑。

### 5.2 右键菜单

仅自定义书架响应右键。打开菜单时只把该书架设为管理目标，不修改 `activeCategory`，因此右侧书籍筛选保持不变。用户只有左键选择书架时才切换筛选。

菜单项目：

1. **重命名**：进入当前行的行内编辑状态。
2. **编辑其他语言名称…**：打开现有多语言名称编辑界面，保留已有翻译能力。
3. 分隔线。
4. **删除书架…**：危险操作，使用危险色并打开确认对话框。

菜单通过以下方式关闭：选择操作、点击菜单外部、按 Esc，或切换页面。

### 5.3 行内重命名

- 输入框默认选中当前语言下显示的完整名称。
- Enter 保存，Esc 取消，失焦时保存非空且不重复的值，否则恢复编辑状态并提示原因。
- 保存期间禁用重复提交。
- 保存时沿用现有级联逻辑，同步更新分类记录、使用旧分类名称的书籍记录，以及当前语言的翻译记录。
- 保存成功后名称原位更新；如果重命名的是当前选中书架，则同步更新 `activeCategory`，使右侧筛选保持在同一书架。
- 数据库写入失败时保留输入内容并显示 toast，不静默回退。

### 5.4 删除书架

确认对话框必须同时显示：

- 将删除的书架名称。
- 该书架当前包含的书籍数量。
- 明确说明书籍不会被删除，而会移动到“待整理”。

示例：

> 删除书架“技术”？其中 7 本书不会被删除，将移至“待整理”。

确认删除后：

1. 将该书架下的书籍分类更新为现有未分类存储值。
2. 删除分类记录及对应翻译记录。
3. 如果被删除书架当前处于选中状态，切换到“待整理”；否则保持现有筛选。
4. 刷新分类数量并显示成功 toast。

任一步骤失败时显示错误提示，并重新加载书籍和分类数据，避免界面状态与数据库不一致。

## 6. 组件边界

为避免继续扩大 `Books.tsx` 中的内联 JSX，实施时将侧栏拆分为聚焦组件：

- `BookCategorySidebar`：负责两个分组、固定入口和新增入口的布局。
- `BookCategoryRow`：负责自定义书架的默认、悬停、选中和行内编辑状态。
- `CategoryContextMenu`：负责右键菜单定位、键盘关闭和菜单操作。
- `DeleteCategoryDialog`：负责显示影响范围并确认删除。

`Books.tsx` 继续持有分类、书籍、翻译及当前筛选状态，并通过回调向侧栏组件提供新增、重命名、编辑翻译和删除行为。此次不重构阅读器或书籍卡片区域。

## 7. 数据与兼容性

- 不新增或修改 SQLite 表。
- `activeCategory` 继续使用稳定的系统值 `all` 和 `uncategorized` 代表两个固定入口。
- 自定义书架暂时继续使用现有分类名称作为筛选值，实施时必须确保重命名后同步更新当前选中状态。
- “待整理”是展示文案变化，不改变数据库已有的未分类存储值。
- 继续识别现有未分类别名、空分类和指向不存在分类的书籍。
- 保留分类翻译表及当前多语言编辑能力。

## 8. 文案与本地化

需要为中文和英文补充或调整以下文案：

- 全部书籍 / All Books
- 待整理 / To Organize
- 我的书架 / My Shelves
- 重命名 / Rename
- 编辑其他语言名称… / Edit Other Language Names…
- 删除书架… / Delete Shelf…
- 删除确认说明及书籍数量插值
- 新增、重命名的重复名称与空名称错误提示

“未分类 / Uncategorized”仍可用于数据兼容和导入表单中的归属含义；左侧快速入口使用新的“待整理 / To Organize”文案键，避免全局替换影响其他场景。

## 9. 可访问性与桌面行为

- 自定义行应支持 `contextmenu` 事件，同时阻止浏览器默认菜单。
- 右键菜单打开后，焦点移入菜单；上下方向键移动，Enter 执行，Esc 关闭并把焦点还给原书架行。
- 菜单项使用语义化按钮或菜单角色。
- 新增与行内重命名输入框提供清晰的可访问名称。
- 固定入口和自定义书架都可通过键盘聚焦并按 Enter 选择。
- 删除仍需明确的确认对话框，不提供键盘快捷删除。

## 10. 测试与验收

### 10.1 功能测试

- “全部书籍”显示所有书籍，数量正确。
- “待整理”包含空分类、兼容别名和失效分类书籍，数量正确。
- 新增书架支持 Enter 提交、Esc 取消、重复名称校验。
- 左键选择自定义书架能正确筛选书籍。
- 只有自定义书架能打开右键菜单。
- 右键不会意外触发书架筛选切换。
- 重命名成功后数据库、当前筛选、书籍分类和翻译显示保持一致。
- 编辑其他语言名称的现有能力仍可使用。
- 删除确认显示正确的书架名称和书籍数量。
- 删除后书籍进入“待整理”，书籍记录不丢失。
- 删除当前书架后自动切换到“待整理”；删除非当前书架不改变筛选。
- 数据库操作失败时界面能恢复到重新加载后的真实状态。

### 10.2 视觉与交互验收

- 默认状态下没有编辑或删除图标。
- 悬停、选中和右键菜单打开时，书架名称与数量不发生位移。
- 长书架名称使用省略号，数量和菜单不被挤出侧栏。
- 右键菜单不会超出窗口边缘；靠近底部时向上展开。
- 浅色、深色、卡片和暗黑科技主题下均保持可读对比度。
- 中英文切换后分组名称、菜单和确认文案完整显示。

## 11. 非目标

- 不实现多级嵌套书架。
- 不实现拖拽排序或拖拽移动书籍。
- 不修改书籍卡片、阅读器或导入流程的整体设计。
- 不改变书籍分类的数据库模型。
- 不为固定入口提供隐藏、排序或删除能力。
